What is OpenDesign

A design tool your coding agent can drive.

OpenDesign is an open-source Figma alternative built for AI workflows. Describe a screen, and your agent writes an HTML file to disk. OpenDesign watches the filesystem, renders it as an iframe, and pins it to an infinite canvas.

No accounts. No cloud. No proprietary format. Your designs are files on your machine — version them with git, edit them with vim, review them in a PR.

01 — Open Source

MIT licensed. Fork it, extend it, own it.

Every line of code is on GitHub. No telemetry, no usage limits, no "free tier." You run it, you control it.

02 — Filesystem-Native

Your designs are files. Always were.

Each frame is a plain HTML file in a project directory. The manifest is JSON. The layout is JSON. cat it, grep it, git diff it.

03 — Agent-Agnostic

If it can write a file, it can design.

OpenDesign works with any coding agent. Not locked to one model or one provider. Bring whatever tool you already use.

On Disk

This is what a project looks like.

No binary blobs. No cloud sync. Just a directory you can ls.

my-app/ ├── PROJECT.md # what you're building ├── DESIGN.md # design tokens (colors, type, spacing) ├── FEEL.md # motion, composition, vibe ├── frames.json # manifest — [{id, name, file}] ├── dashboard.html # a frame — self-contained HTML ├── login.html ├── pricing.html └── .opendesign/ └── layout.json # canvas positions (x, y, w, h)
Bring Your Own Agent

Works with any coding agent.

OpenDesign watches your project directory. Any agent that can write an HTML file — or hit a local HTTP endpoint — can put a frame on the canvas. No plugins, no integrations, no setup.
